O que são APIs?

API (Application Programming Interface) é um conjunto de rotinas e padrões estabelecidos por um software para a utilização das suas funcionalidades por aplicativos que não pretendem envolver-se em detalhes da implementação do software.

Segurança por Design em APIs

As APIs oferecem segurança por design porque sua posição como intermediário entre dois sistemas separa o aplicativo do serviço que fornece a funcionalidade (IBM CLOUD EDUCATION, 2020).

O terminal da API atua como uma barreira de segurança, controlando o acesso e a comunicação entre os sistemas.

Desenvolvimento Baseado em Componentes

Segundo Eler (2006), o desenvolvimento de software baseado em componentes é um conjunto de práticas usadas para construir sistemas de software a partir de componentes reutilizáveis, tanto por terceiros quanto pelo próprio desenvolvedor do sistema.

Objetivo Principal

Adaptar, integrar e atualizar componentes de software reutilizáveis, permitindo que os desenvolvedores se concentrem na lógica de negócio em vez de detalhes de implementação.

O que são Frameworks?

Framework é uma estrutura de suporte definida na qual um outro projeto de software pode ser organizado e desenvolvido. Geralmente inclui programas de suporte, bibliotecas de código, linguagens de script e outros softwares.

Características dos Frameworks

  • Inversão de Controle: O framework chama o código do desenvolvedor, não o contrário
  • Padronização: Define uma estrutura padrão para o desenvolvimento
  • Extensibilidade: Permite personalização através de pontos de extensão

Diferenças entre Frameworks, APIs e Componentes

As diferenças existentes entre essas ferramentas estão relacionadas ao ambiente de programação, ao software desenvolvido e às funcionalidades oferecidas.

Ambiente de Programação

  • Frameworks: Definem a arquitetura e o fluxo da aplicação
  • APIs: Oferecem interfaces para comunicação entre sistemas
  • Componentes: São blocos de construção reutilizáveis

Atividade Prática: Questão 1

Código Java:

int idade = 18;
if (idade >= 18) {
    System.out.println("Maior de idade");
} else {
    System.out.println("Menor de idade");
}

Questão: O que será impresso no console?

A) Erro de compilação

B) Nada será impresso

C) Menor de idade

D) Maior de idade

E) 18

Análise e Resposta:

A variável idade é inicializada com valor 18.

A condição idade >= 18 é verdadeira (18 >= 18).

Portanto, será executado o bloco if, imprimindo "Maior de idade".

Resposta correta: D) Maior de idade

Atividade Prática: Questão 2

Código:

for (int i = 1; i <= 3; i++) {
    System.out.print(i + " ");
}

Questão: O que será impresso no console?

A) 0 1 2

B) 1 2 3

C) 1 2 3 4

D) 0 1 2 3

E) 2 3 4

Análise e Resposta:

O loop for inicia com i = 1.

A condição é i <= 3, então o loop executará para i = 1, 2, 3.

A cada iteração, imprime o valor de i seguido de espaço.

Resultado: "1 2 3 "

Resposta correta: B) 1 2 3

Atividade Prática: Questão 3

Código:

int i = 1;
while (i <= 3) {
    System.out.print(i + " ");
    i++;
}

Questão: Qual será a saída do seguinte código?

A) 1 2 3

B) 1 2 3 4

C) 0 1 2

D) 1 2

E) Erro de compilação

Análise e Resposta:

O while executa enquanto i <= 3.

Inicialmente i = 1.

Iterações:

  • i=1: imprime "1 ", incrementa para 2
  • i=2: imprime "2 ", incrementa para 3
  • i=3: imprime "3 ", incrementa para 4

Resultado: "1 2 3 "

Resposta correta: A) 1 2 3

Atividade Prática: Questão 4

Código:

int i = 5;
do {
    System.out.print(i + " ");
    i++;
} while (i < 7);

Questão: O que o seguinte código imprimirá?

A) 5

B) 5 6

C) 5 6 7

D) Nada será impresso

E) Loop infinito

Análise e Resposta:

O do-while executa pelo menos uma vez.

Primeira execução: imprime "5 ", incrementa i para 6.

Verifica condição: 6 < 7 é verdadeiro.

Segunda execução: imprime "6 ", incrementa i para 7.

Verifica condição: 7 < 7 é falso, encerra.

Resultado: "5 6 "

Resposta correta: B) 5 6

Atividade Prática: Questão 5

Código:

int dia = 1;
switch (dia) {
    case 1:
        System.out.println("Domingo");
        break;
    case 2:
        System.out.println("Segunda");
        break;
    default:
        System.out.println("Outro dia");
}

Questão: O que será impresso pelo seguinte código?

A) Domingo

B) Segunda

C) Outro dia

D) Terça

E) Nada será impresso

Análise e Resposta:

A variável dia tem valor 1.

O switch avalia o valor e encontra o case 1.

Executa o bloco do case 1: imprime "Domingo".

O break impede a execução dos demais casos.

Resposta correta: A) Domingo

Conclusão

O que Aprendemos:

  • APIs oferecem segurança por design, atuando como intermediárias entre sistemas
  • O desenvolvimento baseado em componentes promove reutilização e produtividade
  • Frameworks definem a estrutura e o fluxo das aplicações
  • A escolha entre frameworks, APIs e componentes depende do contexto e requisitos
  • Praticar com exercícios é essencial para fixar os conceitos
